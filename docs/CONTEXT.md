# Coding Standards
## Principles
- KISS (keep it small & simple)
- Encapsulation OOP principle.
- Single Responsibility principle.
- Open/Closed principle.
- DRY - Do no repeat yourself, this applies to both duplicate code and logic.

## Rules
- Make sure utilize the resources I provided for best practices.
- use early return technique to avoid deep nesting.
- 2-3 max nesting depth but less is better.
- add helpful comments (the what and the why).
- Follow security best practices from OWASP cheatsheet.
- add appropriate console logs on both backend and frontend.



## Don't do / Avoid
- deep nesting
- inheritance
- using other libraries/frameworks.
- deep nesting 
- anti-patterns
